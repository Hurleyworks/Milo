🐛 Critical Issues & Solutions

Issue 1: Pipeline Link State Invalid

Problem: "Pipeline has not been linked yet" error despite calling link()

Root Cause: We were calling setupRayTypes() AFTER pipeline->link(), which modified the pipeline and invalidated the link state.

❌ Before (Incorrect)
// 7. Link pipeline
pipeline->optixPipeline.link(maxTraceDepth);

// 8. Setup ray types and miss programs
setupRayTypes(data.entryPoint, data.numRayTypes);
✅ After (Correct)
// 7. Setup ray types and miss programs (BEFORE linking)
setupRayTypes(data.entryPoint, data.numRayTypes);

// 8. Link pipeline (AFTER all programs are set)
pipeline->optixPipeline.link(maxTraceDepth);

Issue 2: Hit Group SBT Not Set

Problem: OptiX requires a valid hit group SBT even for empty scenes

Solution: Always allocate at least 1 byte for hit group SBT, set it immediately after linking.

// Always allocate at least 1 byte (OptiX requirement)
size_t bufferSize = std::max(hitGroupSbtSize, 1);

pipeline->hitGroupSBT.initialize(
    cuContext, cudau::BufferType::Device, bufferSize, 1);
pipeline->hitGroupSBT.setMappedMemoryPersistent(true);

// Set hit group SBT on pipeline
pipeline->optixPipeline.setHitGroupShaderBindingTable(
    pipeline->hitGroupSBT,
    pipeline->hitGroupSBT.getMappedPointer());

🏗️ Correct OptiX Pipeline Setup Sequence

Critical: The order of operations is crucial for OptiX pipeline setup!
  1. Create Pipeline Object - optixContext.createPipeline()
  2. Set Pipeline Options - Configure payload, attributes, traversal flags
  3. Load Module - Create module from PTX/OptiX IR
  4. Create Programs - Ray gen, miss, hit groups
  5. Setup Ray Types - setNumMissRayTypes(), setMissProgram()
  6. Link Pipeline - pipeline.link(maxTraceDepth)
  7. Set Scene - Associate with acceleration structure
  8. Generate SBT Layout - generateShaderBindingTableLayout()
  9. Allocate & Set SBT - Main and hit group SBTs
  10. Launch - Ready for ray tracing
⚠️ Never modify pipeline after linking! Any changes to programs, ray types, or configuration after linking will invalidate the pipeline state.

💡 Key Lessons Learned

Lesson 1: OptiX Pipeline State Management

  • OptiX pipelines have strict sequencing requirements
  • All configuration must be complete BEFORE linking
  • The pipeline's internal pipelineLinked flag is managed by OptiX
  • Modifying a pipeline after linking silently invalidates it

Lesson 2: SBT Requirements

  • OptiX always requires a valid hit group SBT, even for empty scenes
  • Minimum allocation of 1 byte is necessary
  • SBT must be set after scene association
  • Use setMappedMemoryPersistent(true) for SBT buffers

Lesson 3: Debugging OptiX Issues

  • OptiX error messages can be misleading - "Pipeline not linked" might mean it was invalidated
  • Always check the sequence of operations, not just individual calls
  • Enable verbose logging during development to track state changes
  • The OptiX compiler output confirms successful linking

Lesson 4: Code Simplicity

  • Start simple, add complexity only when needed
  • Production code patterns exist for good reasons
  • Over-engineering leads to harder debugging
  • Clean, simple code is easier to maintain and debug
