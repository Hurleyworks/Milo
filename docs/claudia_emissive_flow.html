Important: A material is only considered emissive if BOTH conditions are met:

5. Pipeline Stage 4: Light Probability Computation

compute_light_probs.cu::computeTriangleImportance

framework/claude_core/excludeFromBuild/common/gpu_kernels/compute_light_probs.cu:22-46

The GPU kernel computes importance values for each triangle based on emission:

CUDA_DEVICE_FUNCTION float computeTriangleImportance(
    GeometryInstanceData* geomInst, uint32_t triIndex,
    const MaterialData* materialDataBuffer) {
    
    const MaterialData &mat = materialDataBuffer[geomInst->materialSlot];
    const Triangle &tri = geomInst->triangleBuffer[triIndex];
    
    // Get triangle vertices
    const Vertex (&v)[3] = {
        geomInst->vertexBuffer[tri.index0],
        geomInst->vertexBuffer[tri.index1],
        geomInst->vertexBuffer[tri.index2]
    };
    
    // Calculate triangle area
    Normal3D normal(cross(v[1].position - v[0].position, 
                         v[2].position - v[0].position));
    float area = 0.5f * length(normal);
    
    // Sample emittance at triangle vertices
    RGB emittanceEstimate(0.0f, 0.0f, 0.0f);
    emittanceEstimate += RGB(getXYZ(tex2DLod<float4>(mat.emittance, 
                                                      v[0].texCoord.x, 
                                                      v[0].texCoord.y, 0)));
    emittanceEstimate += RGB(getXYZ(tex2DLod<float4>(mat.emittance, 
                                                      v[1].texCoord.x, 
                                                      v[1].texCoord.y, 0)));
    emittanceEstimate += RGB(getXYZ(tex2DLod<float4>(mat.emittance, 
                                                      v[2].texCoord.x, 
                                                      v[2].texCoord.y, 0)));
    emittanceEstimate /= 3;  // Average of three vertices
    
    // Calculate importance: luminance √ó area
    float importance = sRGB_calcLuminance(emittanceEstimate) * area;
    return importance;
}

Importance Calculation:

  • Step 1: Sample emission texture at each vertex UV coordinate
  • Step 2: Average the three emission values
  • Step 3: Convert to luminance using sRGB formula
  • Step 4: Multiply by triangle area for final importance

7. Light Distribution Building

Hierarchical Distribution Structure

The importance values computed for emissive triangles are used to build a hierarchical probability distribution:

  1. Triangle Level: Each emissive triangle gets an importance value based on emittance √ó area
  2. Geometry Instance Level: Sum of all triangle importances in a geometry instance
  3. Instance Level: Geometry instance importance scaled by instance transform
  4. Scene Level: Distribution over all emissive instances
// Triangle distribution for a geometry instance
GeometryInstanceData::emitterPrimDist  // Distribution over triangles

// Scene-level distribution

8. Key Insights and Optimizations

Design Decisions

  • Separated Color and Strength: The engine separates emission color from strength, allowing for independent control and optimization
  • Vertex Sampling: Triangle importance is estimated by sampling at vertices rather than integrating over the surface
  • Area Weighting: Larger triangles contribute more to the light distribution, ensuring proper importance sampling
  • Hierarchical Structure: The multi-level distribution allows efficient sampling at different granularities

Performance Considerations

  • Texture Caching: Using CUDA texture objects provides hardware-accelerated caching
  • Parallel Computation: Each triangle's importance is computed independently in parallel
  • Mip-Level 0: The kernel uses mip-level 0 for precise emission values
  • Pre-computed Distributions: Light distributions are pre-computed and updated only when geometry changes
