🚀 RiPRSceneHandler Refactoring Plan

Objective: Refactor RiPRSceneHandler to use the generic SceneHandler for IAS management while maintaining RiPR-specific functionality.

📊 Current State Analysis

What RiPRSceneHandler Currently Does

  • IAS Management: Creates and manages optixu::InstanceAccelerationStructure
  • Instance Creation: Creates instances from RenderableNode objects
  • Buffer Management: Manages instance buffers, IAS memory, and scratch buffers
  • Node Tracking: Maintains NodeMap to track instances by index
  • Selection System: Handles material selection/deselection
  • Motion Updates: Updates transforms for animated objects
  • Instance Data: Manages per-instance data buffers
  • Expired Node Cleanup: Removes expired weak references
  • Light Distribution: Manages emissive instances for lighting
  • Deformation: Handles vertex deformation for animated meshes

🎯 Refactoring Strategy

Core Principle: Separation of Concerns

Delegate generic IAS operations to SceneHandler while keeping RiPR-specific logic in RiPRSceneHandler.

What Moves to Generic SceneHandler

  • IAS creation and configuration
  • Instance add/remove operations
  • IAS rebuild/update logic
  • Traversable handle management
  • Buffer allocation for IAS operations
  • Dirty state tracking
  • Basic instance transform updates

What Stays in RiPRSceneHandler

  • RenderableNode to Instance conversion
  • NodeMap management (instance index to node mapping)
  • Selection material system
  • Instance data buffer management
  • Emissive instance tracking
  • Light distribution building
  • Deformation operations
  • Physics phantom handling
  • Expired node cleanup logic
  • Batch instance creation optimization

Update Constructor

Initialize SceneHandler and configure it for RiPR's needs (fast build, allow updates).

RiPRSceneHandler::RiPRSceneHandler(RenderContextPtr ctx) : ctx(ctx) {
    sceneHandler_ = ctx->getHandlers().sceneHandler;
    sceneHandler_->setConfiguration(
        optixu::ASTradeoff::PreferFastBuild,
        true,   // allowUpdate
        false   // allowCompaction
    );
}

Refactor createInstance Methods

Modify instance creation to use SceneHandler while maintaining node tracking.

void RiPRSceneHandler::createInstance(RenderableWeakRef& weakNode) {
    // Create optixu::Instance from node
    optixu::Instance instance = convertNodeToInstance(weakNode);
    
    // Add to generic handler
    sceneHandler_->addInstance(instance);
    
    // Track in nodeMap
    uint32_t index = sceneHandler_->getInstanceCount() - 1;
    nodeMap[index] = weakNode;
}

Refactor IAS Operations

Replace direct IAS operations with SceneHandler calls.

// Old: ias.rebuild(...)
// New:
void RiPRSceneHandler::rebuildIAS() {
    sceneHandler_->buildIAS();
    travHandle = sceneHandler_->getTraversableHandle();
}

void RiPRSceneHandler::updateIAS() {
    sceneHandler_->updateIAS();
    travHandle = sceneHandler_->getTraversableHandle();
}

Update removeExpiredNodes

Integrate with SceneHandler's remove operations.

uint32_t RiPRSceneHandler::removeExpiredNodes() {
    std::vector<uint32_t> expiredIndices;
    // Find expired nodes...
    
    // Remove from SceneHandler in reverse order
    for (auto it = expiredIndices.rbegin(); it != expiredIndices.rend(); ++it) {
        sceneHandler_->removeInstanceAt(*it);
    }
    
    // Rebuild nodeMap...
    return expiredIndices.size();
}

Handle Motion Updates

Use SceneHandler's transform update capabilities.

bool RiPRSceneHandler::updateMotion() {
    bool updated = false;
    for (const auto& [index, weakRef] : nodeMap) {
        if (auto node = weakRef.lock()) {
            if (node->hasMotion()) {
                float transform[12];
                node->getTransform(transform);
                sceneHandler_->updateInstanceTransform(index, transform);
                updated = true;
            }
        }
    }
    if (updated) {
        sceneHandler_->buildOrUpdateIAS();
    }
    return updated;
}

Clean Up Redundant Code

Remove IAS-related member variables and methods that are now handled by SceneHandler.

  • Remove: optixu::InstanceAccelerationStructure ias;
  • Remove: cudau::Buffer iasMem;
  • Remove: cudau::TypedBuffer<OptixInstance> instanceBuffer;
  • Keep: NodeMap nodeMap; for RiPR-specific tracking

Update Getter Methods

Proxy appropriate calls to SceneHandler.

OptixTraversableHandle RiPRSceneHandler::getHandle() {
    return sceneHandler_->getTraversableHandle();
}

size_t RiPRSceneHandler::getInstanceCount() {
    return sceneHandler_->getInstanceCount();
}

✅ Benefits

Code Reusability: Eliminates duplicate IAS management code across engines
Maintainability: Centralized IAS logic makes updates easier
Performance: Leverages optimized update vs rebuild logic
Consistency: All engines use the same IAS management approach
Separation of Concerns: Clear distinction between generic and engine-specific code

⚠️ Challenges & Solutions

Challenge: Maintaining NodeMap synchronization with SceneHandler's instance list

Solution: Always update NodeMap immediately after SceneHandler operations. Consider adding callbacks or observer pattern if needed.

Challenge: Instance index management when removing instances

Solution: Rebuild NodeMap after bulk removals to ensure correct index mapping.

Challenge: Batch operations optimization

Solution: Add batch methods to SceneHandler if needed, or use existing methods with deferred rebuild.

Challenge: Access to internal IAS for RiPR-specific operations

Solution: Add accessor methods to SceneHandler if absolutely necessary, but prefer using public API.
